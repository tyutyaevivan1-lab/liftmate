# LiftMate

Telegram-бот для трекинга тренировок в зале. Пиши боту в свободной форме о выполненном подходе — он сам распознает упражнение, вес, повторения и подходы (через OpenAI GPT-4o-mini), сохранит в базу и сравнит результат с прошлым разом.

## Как это работает

Есть два равноправных способа ввода — свободный текст и inline-меню (`/exercises`), дальше они сходятся в одну и ту же логику:

1. Пользователь пишет боту, например: `жим лежа 80кг на 8 раз, 3 подхода`, либо выбирает
   упражнение через `/exercises` → категория → упражнение
2. Для текстового ввода бот отправляет сообщение в OpenAI (GPT-4o-mini) вместе с контекстом:
   последними репликами переписки (`history.py`) и последней сохранённой записью пользователя.
   GPT определяет: это новый подход, дополнение/поправка к последнему обсуждаемому подходу
   (например «ещё один подход», «i made one set more»), неоднозначное сообщение или не
   тренировка вовсе. Извлечение упражнения/веса/повторений/подходов работает независимо
   от языка сообщения. Выбор через меню сразу даёт "новый подход" с известным упражнением.
3. Если не хватает веса, повторений или количества подходов — бот **не сохраняет запись**, а вежливо
   переспрашивает недостающее (FSM-состояние `waiting_for_details` из `states.py`
   хранит уже известные данные, пока пользователь не ответит) — это общий шаг для обоих способов ввода
4. Как только все три параметра известны — результат сохраняется (новая запись) либо
   обновляется (если это было дополнение к последней записи) в SQLite базе данных
5. При сохранении новой записи бот сравнивает вес с предыдущей попыткой этого же упражнения
   и отвечает с учётом прогресса; при обновлении — просто дружески подтверждает изменение

### Навигация: меню команд и постоянная клавиатура

- Нативное меню Telegram (список при вводе "/") регистрируется через
  `bot.set_my_commands(commands=..., language_code=...)` в `main.py` — отдельные списки
  для `ru`/`en`/`fr` (на основе языка **Telegram-клиента**, `language_code`, это отдельный
  механизм от `/language`) плюс запасной вариант на английском без `language_code`
- Постоянная Reply-клавиатура (`ReplyKeyboardMarkup`, `resize_keyboard=True`) с кнопками
  📋 Меню упражнений / 🏆 Таблица лидеров / 🌐 Язык — появляется сразу после `/start` и
  остаётся видимой, пока её не заменят новой (например, после смены языка её подписи
  перерисовываются под новый язык отдельным сообщением, т.к. Telegram не позволяет
  прикрепить reply-клавиатуру через редактирование уже отправленного сообщения)
- Нажатие любой из этих кнопок — алиас соответствующей команды (`handle_reply_keyboard_button`
  в `handlers.py`, распознаёт подпись кнопки на любом из трёх языков) и, как и сами команды,
  прерывает любой текущий FSM-диалог уточнения данных

### Язык интерфейса (/language)

Язык ответов бота — это явная настройка пользователя, а не автоопределение по каждому
сообщению:

- Команда `/language` (доступна в любой момент) показывает клавиатуру из трёх кнопок:
  🇷🇺 Русский / 🇬🇧 English / 🇫🇷 Français. Выбор сохраняется в таблице `user_settings`
  (`database.get_user_language` / `set_user_language`)
- `/start` при первом запуске сначала показывает эту же клавиатуру и только после выбора
  языка присылает обычное приветствие; если язык уже выбран — сразу приветствие на нём
- Пока язык явно не выбран (например, для меню `/exercises`, вызванного без `/start`),
  используется запасной вариант — `language_code` из настроек Telegram-клиента
- ВАЖНО: распознавание упражнения/веса/повторений/подходов из свободного текста работает
  одинаково независимо от того, на каком языке написал пользователь, но ОТВЕЧАЕТ бот всегда
  на языке из `user_settings` — GPT в `ai_parser.py` больше не определяет язык по тексту
  сообщения для этой цели

### Меню упражнений (/exercises)

- Верхняя строка — кнопка "🌐 Открыть меню упражнений" с `web_app=WebAppInfo(url=WEBAPP_URL)`:
  открывает задеплоенный Telegram Web App (папка `webapp/`) прямо внутри Telegram
- Ниже — inline-клавиатура категорий (Грудь, Спина, Ноги, Плечи, Бицепс, Трицепс), по 2 кнопки в ряд
- Внутри категории — упражнения по одной кнопке в ряд, плюс "➕ Добавить своё упражнение" и "⬅️ Назад к категориям"
- При выборе упражнения бот показывает последний результат (если есть) и запускает тот же
  FSM-диалог уточнения веса/повторений/подходов, что и при текстовом вводе
- "Добавить своё упражнение" просит название текстом, сохраняет его в таблицу `custom_exercises`
  и дальше показывает его в списке этой категории при следующих открытиях меню
- Каждое встроенное упражнение хранится в базе под английским названием (см. `exercises_data.py`) —
  это единый ключ, чтобы прогресс не терялся при смене языка интерфейса; в кнопках при этом
  показывается локализованное название на выбранном пользователем языке (ru/en/fr)

Отдельно от `/exercises`, при старте бота (`main.py`) глобально устанавливается постоянная
Menu Button (синяя иконка слева от поля ввода) через `bot.set_chat_menu_button(MenuButtonWebApp(...))` —
она тоже открывает `WEBAPP_URL`. Сам адрес Web App задаётся в `config.py` (`WEBAPP_URL`,
можно переопределить через `.env`). Учти: сам Web App (`webapp/`) пока фронтенд-макет без
реальной интеграции с ботом/БД — это следующий шаг.

### Лидерборд (/leaderboard)

- При каждой НОВОЙ записи тренировки (`_save_and_reply` в `handlers.py`) обновляется серия
  дней подряд (`database.update_streak_on_workout`): тренировка вчера — серия +1, тренировка
  сегодня же (повторная в тот же день) — серия не меняется, пропуск дня и больше — серия
  сбрасывается на 1. Рекордная серия (`longest_streak`) обновляется отдельно и не уменьшается
- `/leaderboard` показывает глобальный топ-10 по текущей серии и место пользователя (даже если
  он не в топ-10) — пользователи анонимны, псевдоним "Атлет #XXXX" — последние 4 цифры user_id
  (чистая функция, не хранится отдельно, поэтому всегда одинакова для одного пользователя)
- ЭТАП 1 — только бесплатный глобальный лидерборд по сериям. Архитектура готова к расширению:
  `get_leaderboard_top()`/`get_user_rank()` в `database.py` уже принимают опциональный `user_ids`
  (всё, что понадобится будущим дружеским лидербордам — передать id участников группы);
  `build_leaderboard_message()` принимает `board_type` (пока только `"streak_global"`) — платные
  лидерборды по весу/% прогресса добавятся отдельными ветками; таблица `user_stats` уже содержит
  `is_premium` для их гейтинга

## Требования

- Python 3.10+
- Токен Telegram-бота (получить у [@BotFather](https://t.me/BotFather))
- Ключ OpenAI API (https://platform.openai.com/api-keys)

## Установка и запуск локально

1. Клонируй проект и перейди в его директорию:

   ```bash
   cd LiftMate
   ```

2. Создай виртуальное окружение и активируй его:

   ```bash
   python3 -m venv venv
   source venv/bin/activate  # для Windows: venv\Scripts\activate
   ```

3. Установи зависимости:

   ```bash
   pip install -r requirements.txt
   ```

4. Скопируй `.env.example` в `.env` и заполни своими ключами:

   ```bash
   cp .env.example .env
   ```

   Открой `.env` и укажи:

   ```
   BOT_TOKEN=токен_от_BotFather
   OPENAI_API_KEY=твой_ключ_OpenAI
   ```

5. Запусти бота:

   ```bash
   python main.py
   ```

6. Открой своего бота в Telegram и напиши `/start` — сначала выбери язык интерфейса,
   затем попробуй отправить, например: `присед 100 на 5`, либо `/exercises`, чтобы выбрать
   упражнение через меню. Сменить язык можно в любой момент командой `/language`.

При первом запуске автоматически создастся файл базы данных `liftmate.db` в корне проекта.

### API-сервер для Web App (api.py)

`webapp/` получает реальные данные тренировок через отдельный FastAPI-сервер — это
**отдельный процесс**, независимый от `python main.py` (подробнее о выборе такой
архитектуры — в комментарии в начале `api.py`). Запускается так:

```bash
uvicorn api:app --reload
```

По умолчанию поднимется на `http://127.0.0.1:8000` (для порта 8000 явно, как в
требованиях, можно `uvicorn api:app --reload --port 8000`). Использует тот же файл
`liftmate.db` и тот же `.env` (нужен `BOT_TOKEN` — используется для проверки подписи
Telegram initData, см. ниже), так что запускать нужно из корня проекта, там же, где
обычно запускается бот.

Проверить, что сервер работает:

- Открой в браузере **http://127.0.0.1:8000/docs** — автоматическая интерактивная
  документация FastAPI (Swagger UI), там можно посмотреть оба эндпоинта и их схемы
- Или через curl:
  ```bash
  curl http://127.0.0.1:8000/api/user/12345/exercises/bench%20press/last \
    -H "X-Telegram-Init-Data: <строка из Telegram.WebApp.initData>"
  ```
  Без корректного заголовка `X-Telegram-Init-Data` (или с неверной подписью) сервер
  ответит `401 Unauthorized` — это ожидаемо, см. пояснение про защиту в `api.py`.

## Структура проекта

```
LiftMate/
├── main.py           # точка входа, запуск бота (polling)
├── api.py             # FastAPI-сервер для Web App: история/последний подход + проверка initData
├── config.py         # загрузка токена и ключей из .env
├── database.py       # работа с SQLite (aiosqlite): workouts + custom_exercises + user_settings + user_stats
├── handlers.py        # обработчики команд, текста и inline-кнопок (aiogram Router + FSM)
├── states.py           # FSM-состояния (ожидание недостающих данных / названия своего упражнения)
├── history.py          # последние реплики переписки с пользователем (контекст для GPT)
├── exercises_data.py    # словарь категорий/упражнений (ru/en/fr) для inline-меню
├── keyboards.py         # построение inline-клавиатур (меню, выбор языка) и локализованные тексты
├── leaderboard.py       # сборка сообщения лидерборда и анонимные псевдонимы
├── ai_parser.py       # парсинг текста и генерация дружеского ответа через OpenAI GPT-4o-mini
├── utils.py            # вспомогательные функции (склонение слов "подход"/"день", шаг веса и т.п.)
├── requirements.txt   # зависимости проекта
├── .env.example        # пример переменных окружения
└── README.md
```

Язык ответов бота — тон дружелюбного «бро по залу» — задаётся явным выбором пользователя
через `/language` (см. раздел выше), а не автоопределением по каждому сообщению. При этом
распознавание упражнения/веса/повторений/подходов из текста работает одинаково независимо
от языка ввода.

Короткие дополнения вроде «ещё один подход» или «i made one set more» распознаются в контексте последнего обсуждаемого упражнения (история переписки + последняя запись в БД) и обновляют её, а не создают новую запись.

Вес, повторения и количество подходов обязательны для сохранения записи. Если пользователь укажет не всё — бот перейдёт в FSM-состояние ожидания и переспросит недостающее; ответит правильно даже если пользователь вместо ответа опишет совсем другое упражнение или напишет что-то не по теме (в этом случае ожидание сбрасывается, чтобы бот не завис).
